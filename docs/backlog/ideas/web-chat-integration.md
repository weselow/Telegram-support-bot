# Идея: Веб-чат на сайте

**Дата:** 2025-12-26
**Статус:** Backlog
**Приоритет:** На будущее

## Описание

Добавить возможность общения с поддержкой через онлайн-чат на сайте (не только через Telegram).

## Концепция

- Пользователь открывает виджет чата на сайте
- Сообщения отправляются боту через API
- Бот создаёт топик в супергруппе (как для Telegram-пользователей)
- Ответы сотрудников пересылаются обратно в веб-чат (не в Telegram)

## Архитектурные соображения

Потребуется:
- WebSocket-сервер или polling для веб-чата
- API endpoints для отправки/получения сообщений
- Фронтенд-виджет чата
- Идентификация пользователей (сессия, email?)
- Отдельный channel_type в таблице users (telegram / web)

## Влияние на текущую архитектуру

Для подготовки можно:
- Абстрагировать отправку сообщений пользователю (интерфейс IMessageChannel)
- Хранить channel_type в users

На текущую разработку **не влияет** — можно добавить позже.
